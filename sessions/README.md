# Session Logs

> Append-only session log repository. Each file follows the format defined in `directives/Gemini.md` Section 9.

## Naming Convention

```
YYYY-MM-DD_SC-YYYY-MMDD-NNN.md
```

## What Each Log Contains

- Developer name, branch, module, task
- Change type (Functionality / UI/UX / Bug Fix / Refactor)
- **Planned changes** — what the developer declared at session start
- **Files actually modified** — what git shows was touched
- Commits made during the session
- Boundary confirmation (PASS or HALT)
- QA scores across 4 dimensions

This gives reviewers a clear picture: what was *planned* vs. what *actually happened*.

## Rules

- One file per session
- Append-only — never edit or delete a session log
- Every session (clean completion or halt) produces a log entry
- Generated by `scripts/session-close.sh`
- Log format is defined in `directives/Gemini.md` Section 9
